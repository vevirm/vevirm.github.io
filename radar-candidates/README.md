# Research Radar candidates

This folder is generated by `.github/workflows/research-radar.yml`.

`latest.md` is the file to review after each weekly scan. The scanner is conservative: candidates must first pass a hard publication-type gate and a substance gate before topical scoring.

The gate admits only:

- DOI-bearing scholarly journal articles;
- scholarly books and edited-volume chapters from whitelisted academic publishers;
- institutional, policy, and sector reports from recognised organisations;
- borderline preprints or working papers only when they come from a recognised research group or repository and have enough methodological/substantive signals.

It excludes news, blog posts, vendor/consultancy marketing, press releases, listicles, shallow trend roundups, encyclopedic pages, course materials, and weak/thin outputs.

Tune the gate in `assets/research-radar-config.json`, especially `quality_gate.min_substance_signals`, `quality_gate.min_abstract_words`, the domain whitelists, and the exclusion lists.
